# 知识点 
## home
数据: 只能看到自己site,有权限的数据


栏目
- favorite:加星的都算
- rencent:展示的是具体的元素,比如flow,view,metric
- share with me: 可以共享,不能够拒绝


## virtual connection
- 作用: 控制行级权限,典型是确定某个岗位的用户,能看到什么内容
- 要点:username()是获取site的用户名,不是登录名称
https://www.youtube.com/watch?v=MLOTZ85Jri4&t=1017s


## subscribe,alert,metric
subscribe
- 效果: 订阅的方式,就是去发送PDF和邮件
- 订阅人群: 可以主动给订阅,也可以看到了看板后订阅


alert
- 创建条件: 对于数值轴去创建
- 条件: 现在只能够给出常量的数值


metric
- 看单个指标的变动情况,可以增加环比
- 有点展示鼠标的数值和timeline的味道
- 同样在project中能看到

## schedules jobs task
- schedule:展示的计划任务,好像也不能够配置.数据提取总是schedule
- job: 具体跑的job
- task:要跑的各类项目,通过task里面的action,能够重启任务







## permision
权限的类型: 数据查看,下载之类的.

权限的优先级: 
- site,workbook,view均可设置权限
- 上述权限根据"有权限","无权限","未指定"排优先级,只要有"无权限"就是无权限,如果一直未指定,也是无权限
